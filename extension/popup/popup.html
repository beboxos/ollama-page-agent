Rafraichis la page apres avoir change ce reglage.

Vision (modeles multimodaux)
Envoie aussi une capture d'ecran a chaque etape (ex: gemma3, llava, qwen2-vl). Sans effet avec un modele texte seul.

La premiere option est plus fiable ; la seconde reduit les donnees et le temps de traitement.

Pilotage visuel (beta)
Pour les interfaces canvas sans DOM (ex. TeamViewer Web) : envoie uniquement les images et autorise les clics par coordonnees. Les confirmations suivent le reglage ci-dessous.
Mode lecture seule
L'agent peut lire et defiler, mais ne peut ni cliquer, ni saisir, ni valider.

En mode "Toujours demander", le prompt de confirmation propose aussi "Toujours pour cette page" pour ne plus etre redemande le temps de la page en cours.

Ajoutees a la suite des regles internes de l'agent (format d'action, etc.) sans les remplacer.

Le serveur ne repond pas ? (CORS)

Ollama : autorise cette extension en definissant la variable d'environnement avant de lancer ollama serve :

OLLAMA_ORIGINS=chrome-extension://* ollama serve

Serveur compatible OpenAI (LM Studio, FastFlowLM, vLLM...) : verifie dans ses reglages/documentation qu'il autorise les requetes CORS depuis chrome-extension://*, ou depuis n'importe quelle origine si l'option existe.

Dans les deux cas, clique ensuite sur "Autoriser" ci-dessus et accepte la permission de navigation Chrome.
